# Roadmap

## Sequencing Logic

As Steve on TV should be built in the order that preserves taste.

That means:

1. define the product clearly
2. shape the content model and editorial system
3. build the MVP app around real content
4. add growth and insight loops
5. add delight layers that compound the core
6. test monetization only after the site feels culturally real

Do not invert this order.

## Stage 0: Definition

### Goals

- lock the product thesis
- define scope boundaries
- align voice, content model, design principles, and architecture

### Deliverables

- vision docs
- MVP scope doc
- product definition
- content model
- editorial and design principles
- architecture and repo map

### Exit Criteria

- the repo has a coherent documentation system
- major product decisions are written down
- future implementation can proceed without inventing the product on the fly

### Do Not Build Yet

- polished UI
- database-heavy systems
- community features

## Stage 1: Content Foundation

### Goals

- create the first real corpus
- test the schema and editorial rules on actual entries
- establish collections and taxonomy discipline

### Deliverables

- starter taxonomy definitions
- first batch of Steve entries in repo content files
- initial collection set
- content QA checklist

### Exit Criteria

- at least `75-100` strong entries drafted
- collections prove the editorial model
- schema survives real content without awkward workarounds

### Do Not Build Yet

- advanced personalization
- algorithmic recommendation systems
- broad public submission tooling

## Stage 2: MVP App

### Goals

- launch a working public product around real content
- make exact lookup, browse, and entry pages feel sharp

### Deliverables

- static-first Next.js app
- homepage
- search
- browse
- entry pages
- collection pages
- basic submission flow
- lightweight moderation tooling

### Exit Criteria

- `250+` entries published
- search works for obvious queries
- browse and related links create real rabbit holes
- the site feels editorial and collectible, not generic

### Do Not Build Yet

- user accounts
- comments
- saved lists
- full CMS

## Stage 3: Growth, SEO, and Analytics

### Goals

- improve discoverability
- learn how users move through the archive
- strengthen search and content loops

### Deliverables

- SEO refinements
- analytics dashboards or reporting
- search-failure review workflow
- collection and internal-link tuning

### Exit Criteria

- the team understands top entry points, dead ends, and search gaps
- organic discovery improves without page-spam tactics
- editorial decisions are informed by product data

### Do Not Build Yet

- automated content mills
- mass-generated low-value pages
- ad network optimization

## Stage 4: Delight Layer

### Goals

- deepen the product's identity
- add high-craft touches that amplify the archive

### Deliverables

- richer card treatments
- stronger collection packages
- seasonal or thematic editorial drops
- improved relation modules
- more expressive browse surfaces

### Exit Criteria

- the site feels unmistakably itself
- delight improves discovery rather than distracting from it
- sharability improves because the product feels special

### Do Not Build Yet

- novelty UI experiments that weaken readability
- expensive redesigns before content depth justifies them

## Stage 5: Monetization Experiments

### Goals

- test revenue paths that fit the product's taste
- preserve trust while learning what users value enough to buy

### Deliverables

- collectible or editorial product concepts
- limited merch or print experiments
- sponsorship guidelines
- optional membership experiments

### Exit Criteria

- monetization fits the product voice
- revenue experiments do not cheapen the archive
- the core browsing and editorial experience remains primary

### Do Not Build Too Early

- generic display ads
- aggressive affiliate stuffing
- paywalls on the core archive

## What Not To Build Too Early

- every possible taxonomy page
- generalized platform infrastructure
- broad contributor systems
- machine-generated editorial copy
- "engagement" features with no clear product value

The product should earn complexity, not assume it.
